Les controlleurs

Caster est inspiré de la célèbre architecture de codage MVC (Modele Views Controllers)

Par défaut un controlleur sous le main.py est deja enregistré dans l'application et contient les differentes routes de base du framework.

Par la suite 2 methode s'offrent à vous

1-Continuer a definir dans le fichier main.py les nouvelles routes comme ceci

@main.route('/nomderoute')
def fonctionaexecuter():
return render_template("chemin/vers/le/template.html")

2- Définir un autre fichiers de route

Pour en définir une autre il faudra simplement creer un fichiers .py sous cette architecture pour plus de clarté dans le code Ex:blog_controller.py

  • D'abord importer les Blueprint de flask (from flask import Blueprint)
  • Enregistré le Blueprint sous un nom (dans ce cas blog) blog_controller = Blueprint('blog', __name__)
  • Importer ce nouveau fichier de configurations dans le /controllers/__init__.py (from .controllers.blog_controller import blog_controller)
  • Rendez-vous maintenant dans le fichiers /app/__init__.py puis importer encore une fois le nouveau controlleur pour que l'applicattions le reconnaisse. Pour ce faire ajoutez juste cette ligne from .controllers.blog_controller import blog_controller au dessus de cette ligne from .controllers.main import main dans la boucle while

    • Les controlleurs sans paramètres

    Pour définir un controllerus sans paramètres il suffif juste de ne rien déclarer comme parametre dans la fonction qui suit la définition de la route Ex : @main.route('/') def index(): return render_template("/chemin/vers/votre/template.html")

    Les controllers avec paramètres

    Pour ce fait il vous suffira de definir la fonction avec juste les parametres que vous utiliserez pour le traitement de la page

    Les Vues

    Un controlleur renvoie la vue sous forme de template. Il faut noter que dans le fichiers qui vous servirat de controlleur vous pouver utiliser render_template importer de flask pour afficher une vue. Ou si vous voulez rediriger vers une route render_template(url_for('nom_blueprint.fonction'))

    Vous placerez vos différentes vue dans le dossier /views/templates/votre_nom_de_dossier/index.html.Ainsi pour qu'une vue pointe vers ce fichiers vous utilisez le render_template("/votre_nom_de_dossier/index.html")
